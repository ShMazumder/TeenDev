1. Introduction to Version Control Systems

What is Version Control?

A system that records changes to files over time, allowing you to:

  • ✔ Track modifications
  • ✔ Revert to previous versions
  • ✔ Collaborate efficiently

Types of Version Control

Type Description Example
Local Tracks changes on a single machine RCS, SCCS
Centralized Single server hosts all versions SVN, CVS
Distributed Every user has a full repository copy Git, Mercurial

Why Git?

  • Most widely used VCS (93% of developers)
  • Enables non-linear workflows (branching/merging)
  • Fully decentralized for offline work

2. Git Fundamentals

Key Concepts

  • Repository: Project folder tracked by Git
  • Commit: Snapshot of changes with a message
  • Branch: Isolated line of development
  • Merge: Combining branches
  • Remote: Hosted repository (e.g., GitHub)

Essential Commands

Setup & Basics

git init                  # Initialize new repo
git clone <url>           # Clone existing repo
git status                # View changed files

Making Changes

git add <file>            # Stage specific file
git add .                 # Stage all changes
git commit -m "Message"   # Commit staged changes

Branching

git branch                # List branches
git checkout -b feature   # Create & switch to new branch
git merge feature         # Merge into current branch

Undoing Mistakes

git reset --soft HEAD~1   # Undo last commit (keep changes)
git checkout -- <file>    # Discard file changes
git revert <commit-hash>  # Create undo commit

Pro Tip: Use git log --oneline --graph to visualize your commit history with branch structure.

3. GitHub Collaboration

GitHub Workflow

  1. Fork the repository
  2. Clone your fork locally: 
    git clone https://github.com/your-username/repo.git
  3. Add upstream (original repo): 
    git remote add upstream https://github.com/original-owner/repo.git
  4. Sync with upstream: 
    git fetch upstream
git merge upstream/main

Pull Requests (PRs)

  • Create PRs to propose changes to original projects
  • Resolve merge conflicts if they occur
  • Use GitHub Issues to track bugs/enhancements

Open Source Contribution

  • ✔ Start with good-first-issue labeled tasks
  • ✔ Follow project guidelines (CONTRIBUTING.md)
  • ✔ Keep commits small and focused

4. Practical Task: Branch Creation & Submission

Task Instructions

  1. Fork the test repository:

    https://github.com/ShMazumder/test-examples

  2. Create a branch named after your GitHub username: 
    git checkout -b your-username
  3. Make changes (e.g., add a file with your name)
  4. Push your branch: 
    git push origin your-username
  5. Create a Pull Request to the original repository

Important: Your branch name must exactly match your GitHub username for verification.

5. Best Practices & Workflow

Git Hygiene

  • Write descriptive commit messages (e.g., "Fix login button alignment")
  • Commit often, push logically related changes together
  • Use .gitignore to exclude unnecessary files

Branching Strategies

Strategy Use Case
Feature Branches New functionality (e.g., feat/user-auth)
Hotfix Branches Critical bug fixes
Git Flow Complex projects with versions

CI/CD Integration

Automate testing with GitHub Actions:

name: Node.js CI
on: [push]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - run: npm install && npm test
