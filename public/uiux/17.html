Wireframing & Low-Fidelity Prototypes

📘 Day 17: Wireframing & Low-Fidelity Prototypes – Sketch Before You Build

🎯 Objective:

Understand how to plan the structure and flow of your application using wireframes and low-fidelity prototypes before jumping into high-fidelity design.

🧠 What Are Wireframes?

Wireframes are simple, grayscale layouts that outline:

  • Page structure
  • Placement of UI elements (buttons, headers, images, etc.)
  • Content hierarchy
  • Navigation paths
Think of wireframes as the blueprint of your app.

🧱 Why Wireframing First?

  • Saves time and resources
  • Focuses on functionality, not decoration
  • Encourages early feedback
  • Reduces design clutter

🧰 Types of Wireframes:

Type Detail Level Tools
Sketches (paper)Very lowPen & paper, GoodNotes
Low-FidelityBasic shapesFigma, Balsamiq
Mid-FidelityDetailed layoutFigma, Adobe XD

🔧 Elements of a Wireframe:

  • Headers and navigation bars
  • Image placeholders (use X or grey boxes)
  • Text blocks (lines or lorem ipsum)
  • Buttons (boxes with labels)
  • Forms (fields and labels)

🪜 Wireframing Process (Step-by-Step):

  1. Identify user goals – What task is the user trying to complete?
  2. Sketch primary screens – Login, Home, Dashboard, etc.
  3. Define navigation flow – How do users move between screens?
  4. Add interaction hints – Show how buttons and menus behave

🛠️ Tools You Can Use:

  • 📝 Figma (with wireframe kits)
  • 📐 Balsamiq
  • 📷 Whimsical
  • ✏️ Pen & Paper

🧪 Hands-On Task:

Design wireframes for a simple To-Do app with the following screens:

  • Home screen (task list)
  • Add task screen
  • Task details screen
  • Settings screen

✅ Requirements:

  • Use only rectangles, lines, and basic text
  • No colors, images, or icons
  • Focus on layout and navigation only

📦 Low-Fidelity Prototyping

Once wireframes are done, link them together to simulate flow.

In Figma:

  1. Use the Prototype tab
  2. Connect buttons/links to other frames
  3. Set transitions like “On Tap → Navigate to…”

Test the flow, not the visuals.

✅ Best Practices:

  • Keep it simple and fast
  • Use placeholders instead of real content
  • Annotate important actions (e.g., “Add Task” button)
  • Get early feedback before high-fidelity design

❌ Common Mistakes:

  • Jumping straight to detailed design
  • Ignoring navigation logic
  • Making the wireframe too "pretty"
  • Designing without clear user goals

📝 Summary:

  • Wireframes are foundational to good UI/UX
  • Focus on layout, hierarchy, and flow
  • Use low-fidelity prototypes to test interaction early
