📘 Day 5: User Journey Mapping

🎯 Objective:

Understand how to visualize the entire experience a user goes through when interacting with your product—step-by-step—from discovery to post-use.

📖 Lesson Content:

🧭 What is a User Journey Map?

A User Journey Map is a visual or narrative representation of a user’s interaction with a product or service across different touchpoints. It highlights goals, actions, emotions, and pain points at each stage.

🧩 Key Components of a User Journey Map:

Component Description
Persona Who is this journey about? Use one of your Day 4 personas.
Scenario What task is the user trying to complete?
Stages/Steps What steps does the user take (e.g., from finding the app to completing a task)?
Actions What is the user doing at each stage?
Thoughts/Feelings What are they thinking or feeling at each step?
Pain Points What challenges or frustrations might arise?
Opportunities How can we improve the experience at this step?

🧠 Why Map a User Journey?

  • Identify user frustrations and design opportunities.

  • Discover gaps in the experience.

  • Create empathetic design solutions aligned with user needs.

  • Ensure your app feels intuitive and helpful at each touchpoint.

🎯 Example Scenario: Amina orders groceries using a mobile app

Stage Action Thought Feeling Pain Point Opportunity
Discover Sees ad on Facebook “Could this save me time?” Curious Not sure if app is trustworthy Show trust badges/reviews
Install & Onboard Downloads & signs up “Hope this is fast...” Neutral Slow onboarding Add guest mode/skip option
Browse Products Searches grocery items “Where’s brown rice?” Frustrated Poor search/filtering Add smart search, filters
Add to Cart Adds repeated items “Wish it remembered last time” Tired No saved lists Allow reordering or templates
Checkout Confirms address & pays “I hope it arrives on time” Anxious No delivery tracking Add live tracking, ETA
Post-Purchase Rates the experience “It was okay” Satisfied No loyalty rewards Add points or referral system

📌 Template (Make Your Own)

You can use a table like the one above or draw it in a flowchart with sticky notes or digital tools like:

  • Miro

  • Figma (Journey Mapping template)

  • Lucidchart

  • Canva

🧪 Mini Challenge:

Pick your persona from Day 4 and:

  1. Define a scenario (e.g., booking a doctor’s appointment).

  2. Break it into at least 5 stages.

  3. Fill in the journey map table:

    • Actions

    • Thoughts/Feelings

    • Pain Points

    • Opportunities

📝 Save or draw it out as a journey map. You’ll use this in upcoming wireframe and UI decisions.

🧠 Pro Tip:

User Journey Maps are living documents. As you learn more from user testing or data, update your maps.

💬 Real-World Example: Spotify Free User Journey

Stage Action Thought Pain Point Opportunity
Discover Hears about Spotify “What is Spotify?” Unclear value Simple feature intro
Signup Creates account “Hope it’s easy to use” Long form Social login options
Listen to Music Plays songs “Why ads?” Frequent ads Premium trial upsell
Playlist Create Saves songs “Nice! I can build a list” No offline option Save playlists offline
Exit Closes app “I’ll use it again tomorrow” Nothing retained Push notification setup

📝 Summary:

  • User Journey Mapping helps uncover gaps, frustrations, and moments of delight in your product.

  • It bridges the gap between design and user behavior.

  • A strong journey map = better UX decisions and app flows.
