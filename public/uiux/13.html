üß© What Is Wireframing?

Wireframes are low-fidelity layouts that:

  • Represent structure & layout (not final design)
  • Focus on user flow, hierarchy, and placement
  • Avoid distraction by skipping colors, fonts, images

üöÄ Rapid Prototyping: What & Why?

Prototypes simulate how a user interacts with your product.

  • Connect screens ‚Üí mimic real app behavior
  • Useful for user testing without writing code
  • Get early feedback and iterate fast

üîß Prototyping Tools:

  • Figma (Prototyping tab)
  • Adobe XD
  • InVision
  • Marvel App

üéÆ Hands-On Activity:

  1. Pick a simple flow (e.g., Sign up ‚Üí Dashboard)
  2. Sketch wireframes for:
    • Sign Up
    • Dashboard
    • Profile Page
  3. Link the pages in Figma/Adobe XD
  4. Test the flow by clicking through it

‚úÖ Good Practices for Wireframing & Prototyping

  • Keep it simple and focused
  • Use placeholder text and icons
  • Show only essential functionality
  • Test with real users (friends/classmates!)

‚ùå Common Mistakes to Avoid

  • Adding too much visual detail too early
  • Skipping wireframes and jumping to UI
  • Not testing the prototype with others

üìù Summary:

  • Wireframes help you visualize structure before designing
  • Rapid prototyping lets you test flows early and iterate faster
  • Start low-fidelity ‚Üí build up to high-fidelity
