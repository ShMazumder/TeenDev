2. Git Version Control System Mastery

Basic Git Workflow

  1. Initialize a repo: git init
  2. Stage changes: git add .
  3. Commit changes: git commit -m "Initial commit"

Example: Creating a New Git Repository

mkdir my-project
cd my-project
git init
echo "# My Project" >> README.md
git add README.md
git commit -m "First commit"
Note: Always write meaningful commit messages (e.g., "Fix login button styling" instead of "Update code").

3. Git Command Line and Advanced Techniques

Branching & Merging

git checkout -b feature/login   # Create and switch to a new branch
git merge feature/login      # Merge into main branch

Undoing Changes

git reset --hard HEAD      # Discard all uncommitted changes
git revert <commit-hash>  # Revert a specific commit safely

Rebasing Example

git checkout feature/login
git rebase main           # Apply changes on top of main branch

4. GitHub Collaboration & Open Source

Fork & Pull Request Workflow

  1. Fork a repository on GitHub
  2. Clone your fork: git clone https://github.com/your-username/repo.git
  3. Create a branch: git checkout -b fix-bug
  4. Push changes: git push origin fix-bug
  5. Open a Pull Request (PR) on GitHub

Example: Contributing to Open Source

# Clone your fork
git clone https://github.com/your-username/awesome-project.git

# Add upstream (original repo)
git remote add upstream https://github.com/original-owner/awesome-project.git

# Sync with upstream
git fetch upstream
git merge upstream/main
