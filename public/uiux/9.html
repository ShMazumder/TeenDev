User Testing & Feedback Loop

📘 Day 9: User Testing and Feedback Loop – Design That Works for Real People

🎯 Objective:

Learn how to conduct user testing and build a feedback loop to refine your UI/UX based on real user behavior, not assumptions.

🧠 What is User Testing?

User testing is the process of observing real users as they interact with your app to:

  • Identify usability issues
  • Validate user flows
  • Uncover pain points or confusion
  • Improve overall user satisfaction
🎯 Goal: See how users behave, not just what they say.

👥 Types of User Testing:

Type Description Tools
Moderated TestingTester is guided by a facilitatorZoom, Google Meet
Unmoderated TestingTester explores alone, shares feedbackMaze, Useberry, Lookback
A/B TestingCompares two versions (A vs B)Google Optimize, VWO
First Click TestMeasures where users click firstUsabilityHub

🧪 Test Case Example – Grocery App

Task What You’re Observing
“Find and add milk to your cart”Can they locate dairy? Add it easily?
“Proceed to checkout”Do they struggle with finding the button?
“Apply a coupon”Do they see where to input the code?

✅ How to Conduct a Test (Simple Steps)

  1. Set Clear Goals
    • What do you want to learn?
    • E.g., Can users complete checkout in under 2 minutes?
  2. Prepare a Prototype
    • Use your clickable Figma prototype from Day 8
  3. Create Tasks
    • Give users 3–5 realistic tasks to complete
  4. Recruit 3–5 Users
    • Peers, friends, or target users (non-designers are best!)
  5. Observe Silently
    • Let them narrate while using the prototype
    • Watch where they get confused, stuck, or click wrong
  6. Gather Feedback
    • Ask: “What did you like?” “What was confusing?”
  7. Document Issues
    • Make notes/screenshots or use tools like Loom

🔄 Building the Feedback Loop

User testing is NOT a one-time event!

Repeat the cycle:

  1. Test → Learn → Fix → Retest

Use insights to improve:

  • Button placement
  • Labels/text clarity
  • Navigation paths
  • Visual hierarchy

💡 Tips for Effective Testing

  • Avoid helping users unless necessary
  • Don’t ask leading questions
  • Record the session (with permission)
  • Encourage honest, critical feedback

🧪 Mini Challenge:

  1. Prepare 3 tasks in your prototype.
  2. Ask 2–3 friends or family members to test it.
  3. Observe and document:
    • What they said
    • Where they got stuck
    • Any unexpected behavior

❌ Common Mistakes:

  • Testing only with designers or developers
  • Ignoring negative feedback
  • Overexplaining the app during tests

📝 Summary:

  • User testing = real insight
  • Identify friction points before launch
  • Create a continuous improvement loop using real feedback
